# Haskell Project

## Опис проекту
Цей проект є Haskell-додатком, який взаємодіє з базою даних MySQL. Він має модульну архітектуру, що включає компоненти для роботи з базою даних, моделі даних, командний інтерфейс користувача (CLI) тощо. Проект використовує Docker для запуску бази даних MySQL.

## Компоненти проекту

### 1. `src/`
- **`CLI.hs`**: Реалізує командний інтерфейс користувача.
- **`DAO.hs`**: Відповідає за доступ до даних у базі даних.
- **`DB.hs`**: Містить логіку для підключення до бази даних.
- **`Models.hs`**: Визначає моделі даних, які використовуються в додатку.

### 2. `app/Main.hs`
Головний файл програми. Запускає основну логіку додатка.

### 3. `docker/mysql/init/`
- **`01_schema.sql`**: SQL-скрипт для створення схеми бази даних.
- **`02_seed.sql`**: SQL-скрипт для початкового заповнення бази даних даними.

### 4. `docker-compose.yml`
Файл для налаштування та запуску контейнера Docker. Використовується для запуску MySQL.

## Інструкції по запуску

### 1. Встановлення залежностей
Переконайтеся, що у вас встановлені наступні інструменти:
- [Docker](https://www.docker.com/)
- [Docker Compose](https://docs.docker.com/compose/)
- [GHC (Glasgow Haskell Compiler)](https://www.haskell.org/ghc/)
- [Cabal](https://www.haskell.org/cabal/)

### 2. Запуск бази даних
1. Перейдіть у кореневу директорію проекту:
   ```bash
   cd /haskell-infres
   ```
2. Запустіть Docker Compose:
   ```bash
   docker-compose up --build
   ```
   Це створить і запустить контейнер MySQL. SQL-скрипти `01_schema.sql` і `02_seed.sql` автоматично виконаються для створення схеми та заповнення бази даних.

### 3. Запуск програми
1. Переконайтеся, що база даних працює.
2. Запустіть програму за допомогою Cabal:
   ```bash
   cabal run infres
   ```
   Ця команда запустить додаток і відкриє меню командного інтерфейсу.

### 4. Зупинка бази даних
Щоб зупинити Docker-контейнери, виконайте:
```bash
docker-compose down
```
